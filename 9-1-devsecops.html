DevSecOps Integration

L2 Technical L3 Advanced 15 min read

DevSecOps integrates security practices into the DevOps pipeline, enabling "shift-left" security where vulnerabilities are caught early in development rather than in production. Microsoft Defender for Cloud provides tools for CI/CD pipeline security.

GitHub Actions Integration

YAML (GitHub Actions) 
name: Microsoft Security DevOps

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  security-scan:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Run Microsoft Security DevOps
        uses: microsoft/security-devops-action@v1
        id: msdo
        with:
          # Specify which tools to run
          tools: 'templateanalyzer,terrascan,trivy'
          
      - name: Upload results to Security tab
        uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: ${{ steps.msdo.outputs.sarifFile }}

  container-scan:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Build container image
        run: docker build -t myapp:${{ github.sha }} .

      - name: Scan container image
        uses: azure/container-scan@v0
        with:
          image-name: myapp:${{ github.sha }}
          severity-threshold: HIGH
          run-quality-checks: true

Azure DevOps Integration

YAML (Azure Pipelines) 
trigger:
  - main

pool:
  vmImage: 'ubuntu-latest'

stages:
  - stage: SecurityScan
    displayName: 'Security Scanning'
    jobs:
      - job: MSDO
        displayName: 'Microsoft Security DevOps'
        steps:
          - task: MicrosoftSecurityDevOps@1
            displayName: 'Run Security Scan'
            inputs:
              categories: 'IaC,secrets,code'
              tools: 'templateanalyzer,credscan,eslint'

          - task: PublishPipelineArtifact@1
            inputs:
              targetPath: '$(System.DefaultWorkingDirectory)/.gdn'
              artifact: 'SecurityResults'

      - job: ContainerScan
        displayName: 'Container Image Scan'
        steps:
          - task: Docker@2
            displayName: 'Build image'
            inputs:
              command: 'build'
              dockerfile: 'Dockerfile'
              tags: '$(Build.BuildId)'

          - task: ContainerScan@0
            displayName: 'Scan container'
            inputs:
              imageName: 'myapp:$(Build.BuildId)'
              severityThreshold: 'High'

  - stage: Deploy
    displayName: 'Deploy to Azure'
    dependsOn: SecurityScan
    condition: succeeded()
    jobs:
      - deployment: DeployToAKS
        environment: 'production'
        strategy:
          runOnce:
            deploy:
              steps:
                - script: echo "Deploying..."

Container Image Scanning

Scan container images in CI/CD before pushing to registry:

Bash 
# Using Trivy (open-source scanner)
trivy image --severity HIGH,CRITICAL myapp:latest

# Using Defender for Containers (ACR integration)
# Images pushed to ACR are automatically scanned
az acr build --registry myregistry --image myapp:v1 .

# Check scan results
az acr repository show-manifests \
    --name myregistry \
    --repository myapp \
    --query "[].{Digest:digest,Vulnerabilities:changeableAttributes.quarantineState}"

Block Vulnerable Images

YAML (Azure Policy - AKS) 
# Kubernetes Admission Controller policy
# Block pods with images that have critical vulnerabilities
apiVersion: constraints.gatekeeper.sh/v1beta1
kind: K8sAllowedImages
metadata:
  name: require-scanned-images
spec:
  match:
    kinds:
      - apiGroups: [""]
        kinds: ["Pod"]
  parameters:
    repos:
      - "myregistry.azurecr.io/"
    # Only allow images from ACR that have passed scanning

Key Takeaways
  • Shift-left security catches issues earlier and cheaper
  • Defender for DevOps connects GitHub and Azure DevOps to MDC
  • Microsoft Security DevOps Action provides unified scanning
  • IaC scanning catches misconfigurations before deployment
  • Container scanning prevents vulnerable images in production
  • Use pipeline gates to block high-severity findings
  • Branch protection enforces security checks on PRs
  • Combine pipeline security with runtime MDC protection
