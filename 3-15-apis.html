Defender for APIs

L2 Technical 11 min read

Microsoft Defender for APIs provides full lifecycle protection for APIs published through Azure API Management. Gain visibility into API traffic, detect threats, and improve your API security posture.

Requires API Management

Defender for APIs works with Azure API Management service. APIs must be published through APIM.

Key Capabilities

Capability Description
API Discovery Automatic inventory of all APIs in API Management
Security Insights Identify APIs exposing sensitive data
Posture Assessment Recommendations for API configuration
Threat Detection Runtime alerts for suspicious API activity
OWASP Coverage Detection for OWASP API Top 10 risks

Enable Defender for APIs

  1. Navigate to Microsoft Defender for Cloud
  2. Go to Environment settings
  3. Select your subscription
  4. Find APIs in the Defender plans list
  5. Toggle to On
  6. Click Save
Azure CLI 
# Enable Defender for APIs
az security pricing create \
    --name "Api" \
    --tier "Standard"

# Verify status
az security pricing show --name "Api"
HCL 
resource "azurerm_security_center_subscription_pricing" "apis" {
  tier          = "Standard"
  resource_type = "Api"
}

Onboard APIs

Automatic Discovery

Once enabled, Defender for APIs automatically discovers APIs in your API Management instances.

View API Inventory

KQL 
// List all discovered APIs
SecurityResources
| where type == "microsoft.security/apiCollections"
| extend 
    ApiName = properties.displayName,
    ApimService = properties.apimServiceName,
    SensitiveDataExposed = properties.sensitiveDataExposed,
    AuthType = properties.authenticationTypes
| project ApiName, ApimService, SensitiveDataExposed, AuthType

Onboard Specific API Collection

Azure CLI 
# Onboard API collection for protection
az security api-collection create \
    --resource-group "rg-apim" \
    --service-name "my-apim-instance" \
    --collection-name "my-api"

# List API collections
az security api-collection list \
    --resource-group "rg-apim" \
    --service-name "my-apim-instance"

Security Insights

Sensitive Data Exposure

Defender for APIs identifies APIs that may expose sensitive data:

  • Personal identifiable information (PII)
  • Financial data (credit cards, bank accounts)
  • Health information
  • Authentication credentials

Query Sensitive APIs

KQL 
// Find APIs exposing sensitive data
SecurityResources
| where type == "microsoft.security/apiCollections"
| where properties.sensitiveDataExposed == true
| extend 
    ApiName = properties.displayName,
    DataTypes = properties.sensitiveDataTypes,
    Endpoints = properties.numberOfEndpoints
| project ApiName, DataTypes, Endpoints

// APIs without authentication
SecurityResources
| where type == "microsoft.security/apiCollections"
| where properties.authenticationTypes has "None"
| extend ApiName = properties.displayName
| project ApiName, properties.authenticationTypes

Query API Alerts

KQL 
// Get API-related security alerts
SecurityAlert
| where TimeGenerated > ago(7d)
| where ProviderName == "Azure Security Center"
| where AlertType startswith "API_"
| extend 
    ApiName = tostring(ExtendedProperties.ApiName),
    CallerIP = tostring(ExtendedProperties.CallerIpAddress)
| project TimeGenerated, AlertName, Severity, ApiName, CallerIP
| order by TimeGenerated desc

Security Recommendations

Recommendation Impact
APIs should require authentication Prevent unauthorized access
Enable subscription key requirement Control API access
Use OAuth 2.0 for API authentication Modern auth standards
Enable rate limiting Prevent abuse and DoS
Disable anonymous access Require identity for all calls

Query API Recommendations

KQL 
// Get unhealthy API recommendations
SecurityResources
| where type == "microsoft.security/assessments"
| where properties.displayName has "API"
| where properties.status.code == "Unhealthy"
| extend 
    Recommendation = properties.displayName,
    Resource = properties.resourceDetails.Id,
    Severity = properties.metadata.severity
| project Recommendation, Resource, Severity

Integration with API Management

Configure APIM Policies

XML 
<policies>
    <inbound>
        <!-- Require subscription key -->
        <check-header name="Ocp-Apim-Subscription-Key" 
                      failed-check-httpcode="401" 
                      failed-check-error-message="Subscription key required">
        </check-header>
        
        <!-- Rate limiting -->
        <rate-limit-by-key calls="100" 
                          renewal-period="60" 
                          counter-key="@(context.Subscription.Key)" />
        
        <!-- Validate JWT token -->
        <validate-jwt header-name="Authorization" 
                      failed-validation-httpcode="401" 
                      failed-validation-error-message="Unauthorized">
            <openid-config url="https://login.microsoftonline.com/{tenant}/.well-known/openid-configuration" />
            <required-claims>
                <claim name="aud" match="all">
                    <value>api://my-api</value>
                </claim>
            </required-claims>
        </validate-jwt>
    </inbound>
</policies>
