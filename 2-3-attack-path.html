Attack Path Analysis

L2 Technical L3 Advanced Defender CSPM 20 min read

Attack Path Analysis is a premium CSPM feature that uses graph-based analysis to identify potential attack routes through your cloud environment. It answers the critical question: "How could an attacker reach my critical assets?"

Licensing Requirement

Attack Path Analysis requires Defender CSPM plan (not included in free foundational CSPM). This also enables Cloud Security Graph, agentless scanning, and data-aware security posture.

What is Attack Path Analysis?

Traditional security tools identify individual misconfigurations in isolation. Attack Path Analysis connects these findings to show how they could be chained together to compromise critical assets.

How Attack Path Analysis Works

1. Cloud Security Graph Creation

MDC builds a graph of all resources, their configurations, identities, network paths, and data sensitivity classifications.

2. Critical Asset Identification

Automatically identifies high-value targets: databases with sensitive data, key vaults, identity stores, and resources you've tagged as critical.

3. Entry Point Discovery

Identifies internet-exposed resources, public endpoints, and exploitable vulnerabilities that could serve as initial access points.

4. Path Calculation

Uses graph algorithms to find all possible paths from entry points to critical assets, considering permissions, network paths, and vulnerabilities.

5. Risk Prioritization

Ranks paths by exploitability, impact, and number of steps. Provides specific remediation recommendations to break each attack chain.

API & KQL Queries

KQL 
// Query attack paths via Azure Resource Graph
// Run in Azure Resource Graph Explorer
securityresources
| where type == "microsoft.security/attackpaths"
| extend 
    PathName = properties.displayName,
    RiskLevel = properties.riskLevel,
    EntryPoint = properties.entryPointEntityInfo.entityName,
    Target = properties.targetEntityInfo.entityName,
    StepCount = array_length(properties.graphComponent.nodes)
| project 
    PathName,
    RiskLevel,
    EntryPoint,
    Target,
    StepCount
| order by RiskLevel desc
KQL 
// Find recommendations that are part of attack paths
securityresources
| where type == "microsoft.security/assessments"
| where properties.status.code == "Unhealthy"
| where properties.metadata.attackPathsReferencingThisAssessment > 0
| project
    RecommendationName = properties.displayName,
    AttackPathCount = properties.metadata.attackPathsReferencingThisAssessment,
    ResourceId = properties.resourceDetails.Id
| order by AttackPathCount desc

Remediation Prioritization

Attack path analysis helps prioritize remediation by showing which findings have the most impact on your overall risk posture.

Risk Level Description Priority SLA
Critical Direct path to sensitive data with high exploitability Immediate 24-48 hours
High Path requires 2-3 steps, moderate exploitability High 1 week
Medium Longer path (4+ steps) or lower impact target Medium 30 days
Low Theoretical path with low exploitability Low 90 days
Strategic Fix

Look for "choke points" - single fixes that break multiple attack paths. For example, enabling JIT access on one VM might eliminate 5+ different attack paths.

Key Takeaways
  • Attack Path Analysis requires Defender CSPM plan (paid tier)
  • Shows how misconfigurations chain together to reach critical assets
  • Uses Cloud Security Graph to map resources, identities, and data
  • Prioritize fixes that break multiple attack paths ("choke points")
  • Query attack paths via Azure Resource Graph for reporting
  • Focus on Critical/High risk paths first - these have highest exploitability
