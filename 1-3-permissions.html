Permissions and Roles

L1 Fundamentals L2 Technical 10 min read

Microsoft Defender for Cloud uses Azure Role-Based Access Control (RBAC) to manage permissions. Understanding the built-in roles and required permissions helps implement least-privilege access for security teams.

Detailed Permissions

Security Admin Permissions

Can perform:

  • View all MDC data (alerts, recommendations, secure score)
  • Enable/disable Defender plans
  • Configure auto-provisioning
  • Manage security policies
  • Dismiss and suppress alerts
  • Configure workflow automation
  • Export security data
  • Configure email notifications

Cannot perform:

  • Modify non-security resource settings
  • Create or delete Azure resources
  • Assign roles to other users
Security Reader Permissions

Can perform:

  • View security alerts
  • View recommendations
  • View secure score
  • View security policies
  • View regulatory compliance
  • Access Cloud Security Explorer (read)
  • View attack paths

Cannot perform:

  • Dismiss or change alert status
  • Apply recommendations
  • Configure policies or plans
  • Modify any settings
JIT VM Access Permissions
Action Required Role
Configure JIT policy Security Admin or Contributor (on VM)
Request JIT access Reader (minimum) + custom permissions
Approve JIT request Security Admin or custom role

Custom JIT Request Role:

JSON 
{
  "Name": "JIT Access Requester",
  "Description": "Can request JIT VM access",
  "Actions": [
    "Microsoft.Security/locations/jitNetworkAccessPolicies/initiate/action",
    "Microsoft.Security/locations/jitNetworkAccessPolicies/read",
    "Microsoft.Compute/virtualMachines/read"
  ],
  "AssignableScopes": ["/subscriptions/{subscription-id}"]
}

Assigning Roles

Azure CLI 
# Assign Security Reader at subscription level
az role assignment create \
    --assignee "[email protected]" \
    --role "Security Reader" \
    --scope "/subscriptions/YOUR-SUBSCRIPTION-ID"

# Assign Security Admin at management group level
az role assignment create \
    --assignee "[email protected]" \
    --role "Security Admin" \
    --scope "/providers/Microsoft.Management/managementGroups/YOUR-MG-ID"

# Assign Security Reader to a group
az role assignment create \
    --assignee-object-id "GROUP-OBJECT-ID" \
    --assignee-principal-type "Group" \
    --role "Security Reader" \
    --scope "/subscriptions/YOUR-SUBSCRIPTION-ID"

# List current security role assignments
az role assignment list \
    --scope "/subscriptions/YOUR-SUBSCRIPTION-ID" \
    --query "[?contains(roleDefinitionName, 'Security')].{Principal:principalName, Role:roleDefinitionName}" \
    -o table
PowerShell 
# Assign Security Reader
New-AzRoleAssignment `
    -SignInName "[email protected]" `
    -RoleDefinitionName "Security Reader" `
    -Scope "/subscriptions/$subscriptionId"

# Assign Security Admin to a group
$groupId = (Get-AzADGroup -DisplayName "Security Team").Id
New-AzRoleAssignment `
    -ObjectId $groupId `
    -RoleDefinitionName "Security Admin" `
    -Scope "/subscriptions/$subscriptionId"

# List security role assignments
Get-AzRoleAssignment -Scope "/subscriptions/$subscriptionId" | 
    Where-Object { $_.RoleDefinitionName -like "*Security*" } |
    Select-Object DisplayName, RoleDefinitionName, Scope
HCL (Terraform) 
# Security Reader assignment
resource "azurerm_role_assignment" "security_reader" {
  scope                = data.azurerm_subscription.current.id
  role_definition_name = "Security Reader"
  principal_id         = data.azuread_group.soc_analysts.object_id
}

# Security Admin assignment
resource "azurerm_role_assignment" "security_admin" {
  scope                = data.azurerm_subscription.current.id
  role_definition_name = "Security Admin"
  principal_id         = data.azuread_group.security_admins.object_id
}

# Custom JIT role
resource "azurerm_role_definition" "jit_requester" {
  name        = "JIT Access Requester"
  scope       = data.azurerm_subscription.current.id
  description = "Can request JIT VM access"

  permissions {
    actions = [
      "Microsoft.Security/locations/jitNetworkAccessPolicies/initiate/action",
      "Microsoft.Security/locations/jitNetworkAccessPolicies/read",
      "Microsoft.Compute/virtualMachines/read"
    ]
  }

  assignable_scopes = [data.azurerm_subscription.current.id]
}

Recommended Role Structure

Team Role Scope Purpose
Security Leadership Security Admin Management Group Full MDC management across org
SOC Tier 2/3 Security Admin Subscription Investigate and manage alerts
SOC Tier 1 Security Reader Subscription Monitor and triage alerts
Compliance Team Security Reader Management Group Audit and reporting
DevOps Teams Contributor + Security Reader Resource Group Remediate own resources
Developers JIT Requester (custom) Resource Group Request VM access

GCP Connector Requirements

The GCP service account needs these roles:

  • roles/iam.securityReviewer - Read security configurations
  • roles/cloudasset.viewer - View cloud assets
  • roles/compute.viewer - View compute resources

Auditing Role Assignments

KQL 
// Security role assignment changes
AzureActivity
| where OperationNameValue contains "Microsoft.Authorization/roleAssignments"
| where Authorization.scope contains "Security"
| project TimeGenerated, Caller, OperationNameValue, 
          ActivityStatusValue, Properties
| order by TimeGenerated desc
KQL 
// Current Security Admin/Reader assignments
arg("").authorizationResources
| where type == "microsoft.authorization/roleassignments"
| extend roleId = tostring(properties.roleDefinitionId)
| where roleId contains "Security"
| project principalId = properties.principalId, 
          roleDefinitionId = roleId,
          scope = properties.scope

Best Practices

RBAC Implementation Checklist
Key Takeaways
  • Security Reader provides read-only access to all MDC data
  • Security Admin adds configuration and management capabilities
  • Use groups for role assignments, not individual users
  • Assign roles at management group level for org-wide coverage
  • Create custom roles for specific needs like JIT requests
  • Resource Contributor is needed to actually remediate findings
  • Enable Privileged Identity Management (PIM) for just-in-time admin access
  • Regularly audit security role assignments
