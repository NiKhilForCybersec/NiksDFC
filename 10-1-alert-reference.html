Severity Levels

Severity Response Time Description Examples
Critical < 1 hour Active attack, immediate response required Ransomware detected, data exfiltration
High < 4 hours Likely malicious activity requiring investigation Malware execution, privilege escalation
Medium < 24 hours Potentially unwanted or suspicious activity Unusual access patterns, recon activity
Low < 72 hours Informational, may indicate early attack stages Port scanning, anomalous login
Informational As needed No immediate threat, awareness only Security configuration issues

MITRE ATT&CK Mapping

Tactic MDC Detection Coverage
Initial Access SQL injection, brute force, suspicious login, phishing domain access
Execution Suspicious PowerShell, script execution, attack tools
Persistence Startup scripts, scheduled tasks, container modifications
Privilege Escalation Privileged containers, sudo abuse, role assignment changes
Defense Evasion Tor access, log tampering, disabling security tools
Credential Access Mimikatz, credential theft tools, Key Vault access
Discovery AzureHound, MicroBurst, network scanning, API enumeration
Lateral Movement RDP from compromised VM, service account abuse
Collection Data staging, storage access patterns
Exfiltration Unusual data download, DNS tunneling, cloud storage abuse
Impact Ransomware, crypto mining, data destruction

Querying Alerts

KQL 
// All alerts by severity - last 7 days
SecurityAlert
| where TimeGenerated > ago(7d)
| summarize Count = count() by AlertSeverity
| order by Count desc
KQL 
// Alerts mapped to MITRE tactics
SecurityAlert
| where TimeGenerated > ago(30d)
| extend Tactics = todynamic(Tactics)
| mv-expand Tactics
| summarize AlertCount = count() by tostring(Tactics)
| order by AlertCount desc
KQL 
// High/Critical alerts with entity details
SecurityAlert
| where TimeGenerated > ago(24h)
| where AlertSeverity in ("High", "Critical")
| extend Entities = todynamic(Entities)
| mv-expand Entities
| extend EntityType = Entities.Type, EntityName = Entities.Name
| project TimeGenerated, AlertName, AlertSeverity, EntityType, EntityName, Status
Key Takeaways
  • MDC generates alerts for VMs, containers, storage, databases, and more
  • Five severity levels: Critical, High, Medium, Low, Informational
  • All alerts map to MITRE ATT&CK tactics and techniques
  • Critical/High alerts require immediate investigation
  • Alerts include affected resource, evidence, and remediation steps
  • Use KQL to query and analyze alert patterns
  • Integrate with MDE for deeper investigation
  • Configure automation to respond to specific alert types
