Splunk Add-on Configuration

L2 Technical 11 min read

Detailed configuration guide for the Splunk Add-on for Microsoft Cloud Services to ingest Microsoft Defender for Cloud alerts and security data into Splunk.

Prerequisites

  • Splunk Enterprise 8.x or Splunk Cloud
  • Admin access to Splunk
  • Azure subscription with MDC enabled
  • Azure AD app registration with appropriate permissions

Required Azure Permissions

Data Type Required Permission
Security Alerts SecurityEvents.Read.All (Graph API)
Recommendations Security Reader (Azure RBAC)
Secure Score Security Reader (Azure RBAC)
Event Hub Azure Event Hubs Data Receiver

Install Add-on

From Splunkbase

  1. Go to Apps ‚Üí Find More Apps
  2. Search for "Splunk Add-on for Microsoft Cloud Services"
  3. Click Install
  4. Enter Splunk.com credentials
  5. Restart Splunk when prompted

Manual Installation

Bash 
# Download from Splunkbase
cd $SPLUNK_HOME/etc/apps

# Extract the add-on
tar -xvf Splunk_TA_microsoft-cloudservices-*.tgz

# Restart Splunk
$SPLUNK_HOME/bin/splunk restart

Azure App Registration

Create App Registration

Azure CLI 
# Create app registration
az ad app create \
    --display-name "Splunk-MDC-Integration" \
    --sign-in-audience "AzureADMyOrg"

# Get the App ID
APP_ID=$(az ad app list --display-name "Splunk-MDC-Integration" --query "[0].appId" -o tsv)

# Create service principal
az ad sp create --id $APP_ID

# Create client secret (save this!)
az ad app credential reset \
    --id $APP_ID \
    --append \
    --years 2 \
    --query password -o tsv

Assign Permissions

Azure CLI 
# Get service principal object ID
SP_ID=$(az ad sp list --display-name "Splunk-MDC-Integration" --query "[0].id" -o tsv)

# Assign Security Reader role at subscription level
az role assignment create \
    --assignee $SP_ID \
    --role "Security Reader" \
    --scope "/subscriptions/$SUBSCRIPTION_ID"

# For multiple subscriptions, repeat or assign at management group level
az role assignment create \
    --assignee $SP_ID \
    --role "Security Reader" \
    --scope "/providers/Microsoft.Management/managementGroups/$MG_ID"

Grant Graph API Permissions

  1. Go to Azure AD ‚Üí App registrations
  2. Select "Splunk-MDC-Integration"
  3. Click API permissions ‚Üí Add a permission
  4. Select Microsoft Graph ‚Üí Application permissions
  5. Add: SecurityEvents.Read.All
  6. Click Grant admin consent

Configure Add-on

Add Azure Account

  1. In Splunk, go to Settings ‚Üí Data inputs
  2. Click Splunk Add-on for Microsoft Cloud Services
  3. Go to Configuration ‚Üí Account
  4. Click Add
  5. Configure:
    • Account Name: azure-mdc-prod
    • Client ID: (from app registration)
    • Client Secret: (from app registration)
    • Tenant ID: (your Azure AD tenant)
  6. Click Add

Create Security Alerts Input

  1. Go to Inputs tab
  2. Click Create New Input ‚Üí Azure Security Center Alerts
  3. Configure:
    • Name: mdc-alerts
    • Interval: 300 (5 minutes)
    • Index: azure_security
    • Azure Account: azure-mdc-prod
    • Subscription ID: (your subscription or * for all)
  4. Click Add

Input Configuration File

conf 
# inputs.conf
[azure_security_center_alerts://mdc-alerts]
account = azure-mdc-prod
index = azure_security
interval = 300
sourcetype = azure:security:alert
subscription_id = *
start_time = -7d

[azure_security_center_recommendations://mdc-recommendations]
account = azure-mdc-prod
index = azure_security
interval = 3600
sourcetype = azure:security:recommendation
subscription_id = *

[azure_security_center_secure_score://mdc-score]
account = azure-mdc-prod
index = azure_security
interval = 3600
sourcetype = azure:security:score
subscription_id = *

Event Hub Method (Alternative)

For higher throughput and real-time streaming:

Azure Setup

Azure CLI 
# Create Event Hub namespace
az eventhubs namespace create \
    --name "mdc-splunk-eh" \
    --resource-group "security-rg" \
    --sku Standard

# Create Event Hub
az eventhubs eventhub create \
    --name "security-alerts" \
    --namespace-name "mdc-splunk-eh" \
    --resource-group "security-rg" \
    --partition-count 4

# Create consumer group for Splunk
az eventhubs eventhub consumer-group create \
    --name "splunk-consumer" \
    --namespace-name "mdc-splunk-eh" \
    --eventhub-name "security-alerts" \
    --resource-group "security-rg"

# Get connection string
az eventhubs namespace authorization-rule keys list \
    --namespace-name "mdc-splunk-eh" \
    --resource-group "security-rg" \
    --name RootManageSharedAccessKey \
    --query primaryConnectionString -o tsv

Splunk Event Hub Input

conf 
# inputs.conf for Event Hub
[azure_event_hub://mdc-alerts-eventhub]
account = azure-mdc-prod
index = azure_security
sourcetype = azure:security:alert
event_hub_namespace = mdc-splunk-eh.servicebus.windows.net
event_hub_name = security-alerts
consumer_group = splunk-consumer
max_batch_size = 100
interval = 60

Index Configuration

conf 
# indexes.conf
[azure_security]
homePath = $SPLUNK_DB/azure_security/db
coldPath = $SPLUNK_DB/azure_security/colddb
thawedPath = $SPLUNK_DB/azure_security/thaweddb
maxTotalDataSizeMB = 500000
frozenTimePeriodInSecs = 7776000  # 90 days

Recommended Sourcetypes

Data Type Sourcetype Index
Security Alerts azure:security:alert azure_security

Secure Score azure:security:score azure_security

Field Extractions

conf 
# props.conf
[azure:security:alert]
SHOULD_LINEMERGE = false
TIME_FORMAT = %Y-%m-%dT%H:%M:%S
TIME_PREFIX = "startTimeUtc"\s*:\s*"
MAX_TIMESTAMP_LOOKAHEAD = 30
KV_MODE = json
TRUNCATE = 0

# Field aliases for CIM compliance
FIELDALIAS-dest = compromisedEntity AS dest
FIELDALIAS-severity = severity AS severity
FIELDALIAS-signature = alertDisplayName AS signature
FIELDALIAS-vendor_product = "Microsoft Defender for Cloud" AS vendor_product

[azure:security:recommendation]
SHOULD_LINEMERGE = false
KV_MODE = json
TIME_FORMAT = %Y-%m-%dT%H:%M:%S

Validate Configuration

Check Data Ingestion

SPL 
| tstats count WHERE index=azure_security by sourcetype, _time span=1h
| timechart span=1h count by sourcetype

Verify Field Extraction

SPL 
index=azure_security sourcetype="azure:security:alert"
| head 10
| table _time, alertDisplayName, severity, compromisedEntity, status

Check Input Status

SPL 
index=_internal sourcetype=splunkd component=ModularInputs
| search "azure_security_center"
| table _time, log_level, message

Troubleshooting

  • Verify Client ID and Tenant ID
  • Check if client secret has expired
  • Confirm admin consent was granted
  • Verify service principal has required roles
  • Check input is enabled
  • Verify subscription ID is correct
  • Check _internal logs for errors
  • Confirm MDC has alerts to collect
  • Check for multiple inputs configured
  • Verify checkpoint file is working
  • Adjust start_time setting
  • Deduplicate using alert ID

Best Practices

Add-on Configuration Checklist
Key Takeaways
  • Use Splunk Add-on for Microsoft Cloud Services
  • Create dedicated Azure AD app registration
  • Grant Security Reader and Graph API permissions
  • Configure separate inputs for alerts, recommendations, score
  • Use Event Hub for real-time, high-volume data
  • Create dedicated index for Azure security data
  • Monitor _internal for ingestion issues
  • Rotate client secrets before expiration
