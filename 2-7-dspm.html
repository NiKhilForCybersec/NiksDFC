Data Security Posture Management

Defender CSPM L2 Technical 14 min read

Data Security Posture Management (DSPM) in Microsoft Defender for Cloud automatically discovers and classifies sensitive data across your cloud storage resources. It helps identify data exposure risks and prioritize remediation based on data sensitivity.

Requirements
  • Plan: Defender CSPM (premium feature)
  • Extension: Sensitive data discovery must be enabled
  • Supported: Azure Blob Storage, Azure Files, Azure Data Lake Gen2

Enable Sensitive Data Discovery

  1. Azure Portal â†’ Microsoft Defender for Cloud
  2. Environment settings â†’ Select subscription
  3. Defender plans â†’ Defender CSPM â†’ Settings
  4. Enable "Sensitive data discovery"
  5. Configure scan scope (all storage or specific)
Azure CLI 
# Enable Defender CSPM with sensitive data discovery
az security pricing create \
    --name CloudPosture \
    --tier Standard \
    --extensions '[
        {"name": "SensitiveDataDiscovery", "isEnabled": "True"},
        {"name": "ContainerRegistriesVulnerabilityAssessments", "isEnabled": "True"},
        {"name": "AgentlessVmScanning", "isEnabled": "True"}
    ]'

# Verify extensions
az security pricing show --name CloudPosture --query "extensions"
PowerShell 
# Enable Defender CSPM with extensions
$extensions = @(
    @{Name = "SensitiveDataDiscovery"; IsEnabled = "True"},
    @{Name = "AgentlessVmScanning"; IsEnabled = "True"},
    @{Name = "ContainerRegistriesVulnerabilityAssessments"; IsEnabled = "True"}
)

Set-AzSecurityPricing -Name "CloudPosture" -PricingTier "Standard" -Extension $extensions

KQL Query for Sensitive Data Findings

KQL 
// Storage accounts with sensitive data
SecurityResources
| where type == "microsoft.security/assessments"
| where properties.displayName contains "sensitive data"
| extend 
    ResourceId = properties.resourceDetails.id,
    Status = properties.status.code,
    SensitiveTypes = properties.additionalData.sensitiveInfoTypes
| where Status == "Unhealthy"
| project ResourceId, SensitiveTypes
KQL 
// Attack paths involving sensitive data
SecurityResources
| where type == "microsoft.security/attackpaths"
| where properties.attackPathDisplayName contains "sensitive"
| extend 
    PathName = properties.attackPathDisplayName,
    RiskLevel = properties.riskLevel,
    TargetResource = properties.target.azureResourceId
| project PathName, RiskLevel, TargetResource
| order by RiskLevel desc

Remediation Actions

Public Access to Sensitive Data
Azure CLI 
# Disable public blob access
az storage account update \
    --name "storageaccount" \
    --resource-group "rg" \
    --allow-blob-public-access false

# Enable private endpoint
az network private-endpoint create \
    --name "storage-pe" \
    --resource-group "rg" \
    --vnet-name "vnet" \
    --subnet "private-endpoints" \
    --private-connection-resource-id "/subscriptions/.../storageAccounts/storageaccount" \
    --group-id blob \
    --connection-name "storage-connection"
Excessive Access Permissions
Azure CLI 
# Review current role assignments
az role assignment list \
    --scope "/subscriptions/.../storageAccounts/storageaccount" \
    --query "[].{Principal:principalName, Role:roleDefinitionName}" \
    -o table

# Remove overly broad access
az role assignment delete \
    --assignee "[email protected]" \
    --role "Storage Blob Data Contributor" \
    --scope "/subscriptions/.../storageAccounts/storageaccount"

# Assign least-privilege role
az role assignment create \
    --assignee "[email protected]" \
    --role "Storage Blob Data Reader" \
    --scope "/subscriptions/.../storageAccounts/storageaccount/blobServices/default/containers/specific-container"
Missing Encryption
Azure CLI 
# Enable customer-managed keys (CMK)
az storage account update \
    --name "storageaccount" \
    --resource-group "rg" \
    --encryption-key-source Microsoft.Keyvault \
    --encryption-key-vault "https://keyvault.vault.azure.net" \
    --encryption-key-name "storage-key" \
    --encryption-key-version "key-version"

# Require infrastructure encryption (double encryption)
az storage account create \
    --name "newstorage" \
    --resource-group "rg" \
    --require-infrastructure-encryption true

Best Practices

DSPM Implementation Checklist
Key Takeaways
  • DSPM requires Defender CSPM plan with sensitive data discovery extension
  • Automatically discovers PII, PHI, financial data, and credentials
  • Integrates with Cloud Security Graph for attack path analysis
  • Prioritizes risks based on data sensitivity + exposure
  • Works with Microsoft Purview sensitive info types and labels
  • Supports Azure Blob, Files, and Data Lake Gen2
  • Focus remediation on internet-exposed sensitive data first
  • Use least-privilege access and encryption for sensitive stores
