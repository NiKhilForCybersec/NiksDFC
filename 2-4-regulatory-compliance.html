Regulatory Compliance Dashboard

L1 Fundamentals L2 Operations 15 min read

The Regulatory Compliance dashboard in Microsoft Defender for Cloud provides continuous assessment of your Azure resources against regulatory standards and industry benchmarks. Track compliance status, identify gaps, and generate audit-ready reports.

Understanding Compliance Assessment

MDC provides technical control assessment, not compliance certification. The dashboard shows how your Azure configuration aligns with framework requirements, but full compliance requires additional organizational, procedural, and administrative controls.

Adding Compliance Standards

  1. In Regulatory compliance, click "Manage compliance policies"
  2. Select the subscription
  3. Under "Industry & regulatory standards", click "Add more standards"
  4. Select desired standards from the gallery
  5. Click "Add" to assign
Default Standard

Microsoft Cloud Security Benchmark (MCSB) is assigned by default. Add industry-specific standards based on your compliance requirements.

Azure CLI 
# List available compliance standards
az security regulatory-compliance-standards list \
    --query "[].{Name:name,State:state}" -o table

# Assign CIS Azure Foundations
az policy assignment create \
    --name "CIS-Azure-2.0.0" \
    --display-name "CIS Microsoft Azure Foundations Benchmark v2.0.0" \
    --policy-set-definition "/providers/Microsoft.Authorization/policySetDefinitions/06f19060-9e68-4070-92ca-f15cc126059e" \
    --scope "/subscriptions/your-subscription-id"

# Assign PCI DSS v4
az policy assignment create \
    --name "PCI-DSS-v4" \
    --display-name "PCI DSS v4.0" \
    --policy-set-definition "/providers/Microsoft.Authorization/policySetDefinitions/496eeda9-8f2f-4d5e-8dfd-204f0a92ed43" \
    --scope "/subscriptions/your-subscription-id"
PowerShell 
# List regulatory compliance initiatives
Get-AzPolicySetDefinition | 
    Where-Object { $_.Properties.metadata.category -eq "Regulatory Compliance" } |
    Select-Object Name, @{N="DisplayName";E={$_.Properties.DisplayName}} |
    Format-Table -AutoSize

# Assign NIST 800-53 Rev 5
$initiative = Get-AzPolicySetDefinition -Id "/providers/Microsoft.Authorization/policySetDefinitions/179d1daa-458f-4e47-8086-2a68d0d6c38f"

New-AzPolicyAssignment `
    -Name "NIST-800-53-R5" `
    -DisplayName "NIST SP 800-53 Rev. 5" `
    -PolicySetDefinition $initiative `
    -Scope "/subscriptions/$((Get-AzContext).Subscription.Id)"

Managing Exemptions

Exemptions allow you to exclude resources or controls from compliance assessment when you have compensating controls or accepted risk.

Exemption Type Use Case Duration
Waiver Accepted risk, won't remediate Permanent or until expiry
Mitigated Compensating control in place Permanent or until expiry

Create Exemption

Azure CLI 
# Create exemption for a specific resource
az policy exemption create \
    --name "SQL-TDE-Exemption" \
    --display-name "SQL TDE Exemption - Legacy System" \
    --policy-assignment "/subscriptions/.../providers/Microsoft.Authorization/policyAssignments/CIS-Azure-2.0.0" \
    --exemption-category "Waiver" \
    --description "Legacy SQL Server cannot enable TDE. Compensating control: Application-level encryption implemented." \
    --expires-on "2025-12-31" \
    --scope "/subscriptions/.../resourceGroups/legacy-rg/providers/Microsoft.Sql/servers/legacy-sql"
PowerShell 
# Create exemption
New-AzPolicyExemption `
    -Name "NSG-Exemption-DevEnv" `
    -DisplayName "NSG Rule Exemption for Dev Environment" `
    -PolicyAssignment (Get-AzPolicyAssignment -Name "CIS-Azure-2.0.0") `
    -ExemptionCategory "Mitigated" `
    -Description "Dev environment uses Azure Firewall instead of NSG rules" `
    -Scope "/subscriptions/.../resourceGroups/dev-rg" `
    -ExpiresOn (Get-Date).AddYears(1)
Document Exemptions

Always document the business justification for exemptions. During audits, you'll need to explain why controls are exempted and what compensating controls exist.

Compliance Reporting

Export PDF Report

  1. In the Regulatory compliance dashboard, select a standard
  2. Click "Download report" at the top
  3. Select PDF or CSV format
  4. Report includes: compliance percentage, failed controls, affected resources

Programmatic Export

PowerShell 
# Get compliance state for all standards
$subscriptionId = (Get-AzContext).Subscription.Id

# List all standards
$standards = Invoke-AzRestMethod -Method GET `
    -Uri "https://management.azure.com/subscriptions/$subscriptionId/providers/Microsoft.Security/regulatoryComplianceStandards?api-version=2019-01-01-preview"

$standardsList = ($standards.Content | ConvertFrom-Json).value

foreach ($std in $standardsList) {
    Write-Host "$($std.name): $($std.properties.state)" -ForegroundColor Cyan
    
    # Get controls for each standard
    $controls = Invoke-AzRestMethod -Method GET `
        -Uri "https://management.azure.com/subscriptions/$subscriptionId/providers/Microsoft.Security/regulatoryComplianceStandards/$($std.name)/regulatoryComplianceControls?api-version=2019-01-01-preview"
    
    $controlData = ($controls.Content | ConvertFrom-Json).value
    
    $passed = ($controlData | Where-Object { $_.properties.state -eq "Passed" }).Count
    $failed = ($controlData | Where-Object { $_.properties.state -eq "Failed" }).Count
    
    Write-Host "  Passed: $passed | Failed: $failed"
}
KQL 
// Compliance trend over time
SecureScoreControls
| where TimeGenerated > ago(90d)
| summarize 
    Healthy = sum(HealthyResources),
    Unhealthy = sum(UnhealthyResources)
    by bin(TimeGenerated, 1d)
| extend CompliancePercent = round(100.0 * Healthy / (Healthy + Unhealthy), 1)
| project TimeGenerated, CompliancePercent
| render timechart with (title="Compliance Trend - 90 Days")
KQL 
// Top failing controls
SecurityRecommendation
| where RecommendationState == "Unhealthy"
| summarize 
    AffectedResources = count(),
    ResourceList = make_set(AssessedResourceId, 5)
    by RecommendationName, RecommendationSeverity
| order by AffectedResources desc
| take 20

Best Practices

Compliance Management Checklist

Compliance Alert Rule

KQL 
// Alert when compliance drops significantly
SecureScoreControls
| summarize 
    Healthy = sum(HealthyResources),
    Unhealthy = sum(UnhealthyResources)
    by bin(TimeGenerated, 1h)
| extend CompliancePercent = round(100.0 * Healthy / (Healthy + Unhealthy), 1)
| extend PrevCompliance = prev(CompliancePercent)
| where isnotnull(PrevCompliance)
| where CompliancePercent < PrevCompliance - 5  // 5% drop
| project TimeGenerated, CompliancePercent, PrevCompliance, Drop = PrevCompliance - CompliancePercent
Key Takeaways
  • MDC provides technical control assessment - not full compliance certification
  • MCSB is assigned by default; add industry standards as needed
  • Standards are implemented as Azure Policy initiatives
  • Use exemptions for accepted risks with documented justification
  • Export reports regularly for audit evidence
  • Track compliance trends over time with KQL queries
  • Compliance percentage = Passed / (Passed + Failed) * 100
