Secure Score Deep Dive

L1 Fundamentals L2 Technical 20 min read

Secure Score is the primary metric for measuring your cloud security posture in Microsoft Defender for Cloud. It provides a numerical representation (0-100%) of how well your resources are configured according to security best practices.

Score vs Percentage

Secure Score is displayed as both a percentage and points (e.g., "72% - 45/62 points"). The points represent actual vs maximum possible points based on your enabled controls.

Score Calculation Methodology

Understanding how Secure Score is calculated helps you prioritize remediation efforts effectively.

The Formula

Secure Score Formula
Secure Score = (Sum of Earned Points) / (Sum of Maximum Points) Ã— 100% Where for each control: Earned Points = Max Points Ã— (Healthy Resources / Total Resources) Example: Control "Enable MFA" has max 10 points 8 of 10 users have MFA enabled Earned Points = 10 Ã— (8/10) = 8 points

Security Controls

Secure Score is organized into security controls, each representing a group of related security recommendations.

Control Max Points Focus Area Example Recommendations
Enable MFA 10 Identity MFA for admins, MFA for all users
Secure management ports 8 Network Close SSH/RDP, use JIT access
Apply system updates 6 Compute Install OS patches, update agents
Encrypt data in transit 6 Data HTTPS only, TLS 1.2+
Restrict unauthorized access 4 Access Remove external access, limit permissions
Enable endpoint protection 4 Compute Install antimalware, enable MDE
Enable auditing and logging 4 Monitoring Diagnostic settings, audit logs
Pro Tip: Quick Wins

Focus on controls with the highest max points first. "Enable MFA" alone is worth 10 points - more than some entire control categories combined!

Improvement Strategies

Quick Wins (1-2 weeks)

  • Enable MFA for all privileged accounts - Highest impact, relatively easy
  • Enable Microsoft Defender plans - Immediate coverage
  • Restrict management ports - Use NSG rules or JIT access
  • Enable encryption at rest - Most Azure services support this natively

Medium-Term Goals (1-3 months)

  • Remediate vulnerability findings - Patch critical/high CVEs
  • Implement least-privilege access - Review RBAC assignments
  • Enable advanced threat protection - Defender for Servers P2, Containers
  • Configure diagnostic settings - Send logs to Log Analytics

Long-Term Excellence (3-6 months)

  • Implement governance rules - Auto-assign owners to recommendations
  • Create exemption management process - Document risk acceptance
  • Automate remediation - Logic Apps for common fixes
  • Integrate with CI/CD - Shift-left security validation

Tracking Score Over Time

Use KQL queries in Log Analytics or Microsoft Sentinel to track Secure Score trends.

KQL 
// Secure Score trend over last 30 days
SecureScores
| where TimeGenerated > ago(30d)
| summarize 
    AvgScore = avg(PercentageScore),
    MaxScore = max(PercentageScore),
    MinScore = min(PercentageScore)
    by bin(TimeGenerated, 1d)
| order by TimeGenerated asc
| render timechart
KQL 
// Control-level breakdown
SecureScoreControls
| where TimeGenerated > ago(1d)
| summarize arg_max(TimeGenerated, *) by ControlName
| project 
    ControlName,
    CurrentScore = PercentageScore,
    MaxScore,
    HealthyResources,
    UnhealthyResources
| order by MaxScore desc
KQL 
// Compare subscriptions
SecureScores
| where TimeGenerated > ago(1d)
| summarize arg_max(TimeGenerated, *) by SubscriptionId
| project 
    SubscriptionId,
    DisplayName,
    Score = PercentageScore,
    CurrentScore,
    MaxScore
| order by Score asc  // Show lowest scores first
Key Takeaways
  • Secure Score = (Earned Points / Max Points) Ã— 100%
  • Points are earned per control based on healthy vs unhealthy resources
  • Focus on high-point controls first (MFA = 10 points)
  • Use exemptions sparingly and document risk acceptance
  • Track trends over time using KQL in Log Analytics/Sentinel
  • A "good" score depends on your environment, but aim for 70%+ as a baseline
