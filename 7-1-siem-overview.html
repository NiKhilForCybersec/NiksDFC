Third-Party SIEM Integration

L2 Technical L3 Advanced 12 min read

Microsoft Defender for Cloud can stream security alerts and data to third-party SIEM platforms like IBM QRadar, Micro Focus ArcSight, and others using Azure Event Hubs as the integration point.

Integration Methods
  • Event Hub: Real-time streaming (recommended)
  • Graph Security API: REST-based polling
  • Log Analytics Export: Data export rules

Event Hub Configuration

Step 1: Create Event Hub Namespace

Azure CLI 
# Create Event Hub namespace
az eventhubs namespace create \
    --name "mdc-siem-namespace" \
    --resource-group "security-rg" \
    --location "eastus" \
    --sku Standard \
    --enable-auto-inflate true \
    --maximum-throughput-units 20

# Create Event Hub for alerts
az eventhubs eventhub create \
    --name "mdc-alerts" \
    --namespace-name "mdc-siem-namespace" \
    --resource-group "security-rg" \
    --partition-count 4 \
    --message-retention 7

# Create authorization rule for SIEM
az eventhubs eventhub authorization-rule create \
    --name "siem-listen" \
    --namespace-name "mdc-siem-namespace" \
    --eventhub-name "mdc-alerts" \
    --resource-group "security-rg" \
    --rights Listen

# Get connection string
az eventhubs eventhub authorization-rule keys list \
    --name "siem-listen" \
    --namespace-name "mdc-siem-namespace" \
    --eventhub-name "mdc-alerts" \
    --resource-group "security-rg" \
    --query "primaryConnectionString" -o tsv

Step 2: Configure Continuous Export

Azure CLI 
# Create continuous export to Event Hub
az security automation create \
    --name "export-to-siem" \
    --resource-group "security-rg" \
    --scopes '[{"scopePath": "/subscriptions/YOUR-SUBSCRIPTION-ID"}]' \
    --sources '[{"eventSource": "Alerts", "ruleSets": []}]' \
    --actions '[{
        "actionType": "EventHub",
        "eventHubResourceId": "/subscriptions/.../eventhubs/mdc-alerts",
        "connectionString": "YOUR-CONNECTION-STRING"
    }]'

Step 3: Verify Export

Azure CLI 
# Check Event Hub metrics
az monitor metrics list \
    --resource "/subscriptions/.../namespaces/mdc-siem-namespace/eventhubs/mdc-alerts" \
    --metric "IncomingMessages" \
    --interval PT1H

QRadar AQL Query Examples

AQL 
-- MDC High severity alerts last 24 hours
SELECT 
    DATEFORMAT(starttime, 'yyyy-MM-dd HH:mm:ss') as EventTime,
    LOGSOURCENAME(logsourceid) as Source,
    "alertDisplayName" as AlertName,
    "severity" as Severity,
    "compromisedEntity" as Resource
FROM events
WHERE LOGSOURCENAME(logsourceid) ILIKE '%Azure%Defender%'
    AND "severity" IN ('High', 'Critical')
    AND starttime > NOW() - 24 HOURS
ORDER BY starttime DESC
AQL 
-- Alert count by resource type
SELECT 
    "resourceType" as ResourceType,
    COUNT(*) as AlertCount
FROM events
WHERE LOGSOURCENAME(logsourceid) ILIKE '%Azure%Defender%'
    AND starttime > NOW() - 7 DAYS
GROUP BY "resourceType"
ORDER BY AlertCount DESC

Micro Focus ArcSight Integration

ArcSight SmartConnector Configuration

  1. Install Connector: Microsoft Azure Event Hub SmartConnector
  2. Configure Azure Settings:
    Properties 
    # agent.properties
azure.eventhub.namespace=mdc-siem-namespace
azure.eventhub.name=mdc-alerts
azure.eventhub.sas.keyname=siem-listen
azure.eventhub.sas.key=YOUR-SAS-KEY
azure.eventhub.consumer.group=$Default
  3. Map to CEF: Configure field mappings to Common Event Format
  4. Set Destination: ArcSight ESM or Logger

Microsoft Graph Security API

Use the Graph Security API for polling-based integration with any SIEM.

API Authentication

Bash 
# Get access token
curl -X POST "https://login.microsoftonline.com/$TENANT_ID/oauth2/v2.0/token" \
    -H "Content-Type: application/x-www-form-urlencoded" \
    -d "client_id=$CLIENT_ID" \
    -d "client_secret=$CLIENT_SECRET" \
    -d "scope=https://graph.microsoft.com/.default" \
    -d "grant_type=client_credentials"

Query Alerts

Bash 
# Get security alerts
curl -X GET "https://graph.microsoft.com/v1.0/security/alerts_v2?\$top=100&\$filter=severity eq 'high'" \
    -H "Authorization: Bearer $ACCESS_TOKEN" \
    -H "Content-Type: application/json"

Python Integration Example

Python 
import requests
from datetime import datetime, timedelta

class MDCAlertCollector:
    def __init__(self, tenant_id, client_id, client_secret):
        self.tenant_id = tenant_id
        self.client_id = client_id
        self.client_secret = client_secret
        self.token = None
    
    def get_token(self):
        url = f"https://login.microsoftonline.com/{self.tenant_id}/oauth2/v2.0/token"
        data = {
            "client_id": self.client_id,
            "client_secret": self.client_secret,
            "scope": "https://graph.microsoft.com/.default",
            "grant_type": "client_credentials"
        }
        response = requests.post(url, data=data)
        self.token = response.json()["access_token"]
    
    def get_alerts(self, hours=24):
        """Fetch alerts from last N hours"""
        if not self.token:
            self.get_token()
        
        since = (datetime.utcnow() - timedelta(hours=hours)).isoformat() + "Z"
        url = "https://graph.microsoft.com/v1.0/security/alerts_v2"
        params = {
            "$filter": f"createdDateTime ge {since}",
            "$top": 1000,
            "$orderby": "createdDateTime desc"
        }
        headers = {"Authorization": f"Bearer {self.token}"}
        
        response = requests.get(url, headers=headers, params=params)
        return response.json().get("value", [])
    
    def format_for_siem(self, alert):
        """Convert to SIEM-friendly format"""
        return {
            "timestamp": alert.get("createdDateTime"),
            "alert_id": alert.get("id"),
            "title": alert.get("title"),
            "severity": alert.get("severity"),
            "status": alert.get("status"),
            "category": alert.get("category"),
            "description": alert.get("description"),
            "evidence": alert.get("evidence", []),
            "mitre_techniques": alert.get("mitreTechniques", [])
        }

# Usage
collector = MDCAlertCollector(TENANT_ID, CLIENT_ID, CLIENT_SECRET)
alerts = collector.get_alerts(hours=24)
for alert in alerts:
    siem_event = collector.format_for_siem(alert)
    # Send to your SIEM via API/syslog/etc.
    print(siem_event)

Best Practices

SIEM Integration Checklist
Key Takeaways
  • Event Hub provides real-time streaming to any SIEM
  • Graph Security API enables polling-based integration
  • QRadar uses Microsoft Azure DSM with Event Hub connector
  • ArcSight requires SmartConnector for Event Hub
  • Map MDC fields to CEF/LEEF for standardization
  • Use dedicated consumer groups per SIEM instance
  • Monitor Event Hub throughput and lag
  • Python/REST can integrate with any custom SIEM
