Prerequisites

  • ArcSight ESM 7.x or later
  • SmartConnector for Microsoft Azure Event Hub
  • Azure Event Hub namespace
  • Network connectivity from connector to Azure

Azure Configuration

Create Event Hub

Azure CLI 
# Create Event Hub namespace
az eventhubs namespace create \
    --name "mdc-arcsight-eh" \
    --resource-group "security-rg" \
    --location "eastus" \
    --sku Standard

# Create Event Hub
az eventhubs eventhub create \
    --name "defender-alerts" \
    --namespace-name "mdc-arcsight-eh" \
    --resource-group "security-rg" \
    --partition-count 4 \
    --message-retention 3

# Create consumer group
az eventhubs eventhub consumer-group create \
    --name "arcsight-consumer" \
    --namespace-name "mdc-arcsight-eh" \
    --eventhub-name "defender-alerts" \
    --resource-group "security-rg"

# Create SAS policy for ArcSight (Listen only)
az eventhubs eventhub authorization-rule create \
    --name "arcsight-listen" \
    --namespace-name "mdc-arcsight-eh" \
    --eventhub-name "defender-alerts" \
    --resource-group "security-rg" \
    --rights Listen

# Get connection string
CONNECTION_STRING=$(az eventhubs eventhub authorization-rule keys list \
    --name "arcsight-listen" \
    --namespace-name "mdc-arcsight-eh" \
    --eventhub-name "defender-alerts" \
    --resource-group "security-rg" \
    --query primaryConnectionString -o tsv)

echo "Connection String: $CONNECTION_STRING"

Configure Continuous Export

Azure CLI 
# Enable continuous export to Event Hub
az security automation create \
    --name "export-to-arcsight" \
    --resource-group "security-rg" \
    --scopes '[{"scopePath": "/subscriptions/YOUR-SUB-ID"}]' \
    --sources '[{"eventSource": "Alerts"}]' \
    --actions '[{
        "actionType": "EventHub",
        "eventHubResourceId": "/subscriptions/.../eventhubs/defender-alerts",
        "connectionString": "YOUR-CONNECTION-STRING"
    }]'

SmartConnector Configuration

Install SmartConnector

  1. Download SmartConnector for Microsoft Azure Event Hub
  2. Run installer on connector host
  3. Select "Add a Connector"
  4. Choose "Microsoft Azure Event Hub"
  5. Complete installation wizard

Connector Configuration

Properties 
# agent.properties
agents[0].type=azureeventhub
agents[0].name=MDC-Alerts

# Event Hub Settings
agents[0].eventhubnamespace=mdc-arcsight-eh.servicebus.windows.net
agents[0].eventhubname=defender-alerts
agents[0].consumergroup=arcsight-consumer
agents[0].sharedaccesskeyname=arcsight-listen
agents[0].sharedaccesskey=YOUR-SAS-KEY

# Storage for checkpointing
agents[0].storageconnectionstring=YOUR-STORAGE-CONNECTION-STRING
agents[0].storagecontainername=arcsight-checkpoints

# Destination
agents[0].destination.type=cef_tcp
agents[0].destination.host=arcsight-manager.company.com
agents[0].destination.port=9000

Storage Account for Checkpointing

Azure CLI 
# Create storage account for connector checkpointing
az storage account create \
    --name "arcsightcheckpoint" \
    --resource-group "security-rg" \
    --sku Standard_LRS

# Create container
az storage container create \
    --name "arcsight-checkpoints" \
    --account-name "arcsightcheckpoint"

# Get connection string
az storage account show-connection-string \
    --name "arcsightcheckpoint" \
    --resource-group "security-rg" \
    --query connectionString -o tsv

ArcSight Correlation Rules

High Severity MDC Alert

Rule Definition 
Rule Name: MDC High Severity Alert
Condition:
  deviceVendor = "Microsoft" AND
  deviceProduct = "Defender for Cloud" AND
  severity >= 7

Actions:
  - Create Case
  - Send Email to SOC
  - Add to Active List: MDC_Critical_Alerts

Multiple Alerts on Same Host

Rule Definition 
Rule Name: MDC Attack Chain Detection
Type: Aggregation Rule
Condition:
  deviceVendor = "Microsoft" AND
  deviceProduct = "Defender for Cloud"

Aggregation:
  Group By: destinationHostName
  Threshold: 3 events in 1 hour
  Distinct: name (alert types)

Actions:
  - Create High Priority Case
  - Trigger Playbook: Isolate Host

Credential Access Detection

Rule Definition 
Rule Name: MDC Credential Dumping
Condition:
  deviceVendor = "Microsoft" AND
  deviceProduct = "Defender for Cloud" AND
  (name CONTAINS "credential" OR
   name CONTAINS "Mimikatz" OR
   deviceCustomString2 = "CredentialAccess")

Actions:
  - Create Critical Case
  - Send SMS to On-Call
  - Block Host (via firewall integration)

ArcSight Dashboards

MDC Overview Dashboard

  • Alerts by Severity: Pie chart of severity distribution
  • Alert Trend: Line chart over 7 days
  • Top Affected Resources: Bar chart of compromisedEntity
  • MITRE ATT&CK Coverage: Tactic distribution
  • Active Alerts Table: Real-time alert queue

Query Examples

ArcSight Query 
-- High severity alerts last 24 hours
SELECT name, severity, destinationHostName, startTime
FROM events
WHERE deviceVendor = 'Microsoft'
  AND deviceProduct = 'Defender for Cloud'
  AND severity >= 7
  AND startTime > NOW() - 24 HOURS
ORDER BY startTime DESC

-- Alert count by MITRE tactic
SELECT deviceCustomString2 AS tactic, COUNT(*) AS count
FROM events
WHERE deviceVendor = 'Microsoft'
  AND deviceProduct = 'Defender for Cloud'
  AND startTime > NOW() - 7 DAYS
GROUP BY deviceCustomString2
ORDER BY count DESC

Troubleshooting

  • Verify Event Hub connection string
  • Check SAS key hasn't expired
  • Confirm consumer group exists
  • Verify continuous export is enabled
  • Check connector logs for errors
  • Update SmartConnector to latest version
  • Review parser configuration
  • Check sample events in debug mode
  • Create custom FlexConnector if needed
  • Increase Event Hub throughput units
  • Check checkpoint storage performance
  • Scale connector resources
  • Monitor Event Hub metrics
