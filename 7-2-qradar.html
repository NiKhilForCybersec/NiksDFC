Prerequisites

  • QRadar 7.3.2 or later
  • Microsoft Azure DSM installed
  • Azure Event Hub namespace with MDC alerts
  • Network connectivity from QRadar to Azure

Azure Configuration

Create Event Hub

Azure CLI 
# Create Event Hub namespace
az eventhubs namespace create \
    --name "mdc-alerts-qradar" \
    --resource-group "security-rg" \
    --location "eastus" \
    --sku Standard \
    --enable-auto-inflate \
    --maximum-throughput-units 20

# Create Event Hub for alerts
az eventhubs eventhub create \
    --name "defender-alerts" \
    --namespace-name "mdc-alerts-qradar" \
    --resource-group "security-rg" \
    --partition-count 4 \
    --message-retention 7

# Create consumer group for QRadar
az eventhubs eventhub consumer-group create \
    --name "qradar-consumer" \
    --namespace-name "mdc-alerts-qradar" \
    --eventhub-name "defender-alerts" \
    --resource-group "security-rg"

# Create authorization rule for QRadar (Listen only)
az eventhubs eventhub authorization-rule create \
    --name "qradar-listen" \
    --namespace-name "mdc-alerts-qradar" \
    --eventhub-name "defender-alerts" \
    --resource-group "security-rg" \
    --rights Listen

# Get connection string
az eventhubs eventhub authorization-rule keys list \
    --name "qradar-listen" \
    --namespace-name "mdc-alerts-qradar" \
    --eventhub-name "defender-alerts" \
    --resource-group "security-rg" \
    --query primaryConnectionString -o tsv

Configure Continuous Export

Azure CLI 
# Create continuous export to Event Hub
az security automation create \
    --name "export-to-qradar" \
    --resource-group "security-rg" \
    --scopes '[{"scopePath": "/subscriptions/YOUR-SUBSCRIPTION-ID"}]' \
    --sources '[
        {"eventSource": "Alerts"},
        {"eventSource": "Assessments"}
    ]' \
    --actions '[{
        "actionType": "EventHub",
        "eventHubResourceId": "/subscriptions/.../eventhubs/defender-alerts",
        "connectionString": "YOUR-CONNECTION-STRING"
    }]'

QRadar Configuration

Install Azure DSM

  1. Download Microsoft Azure DSM from IBM Fix Central
  2. Go to Admin ‚Üí Extension Management
  3. Click Add and upload the DSM package
  4. Install and deploy the extension

Create Log Source

  1. Go to Admin ‚Üí Log Sources
  2. Click Add
  3. Configure:
    • Log Source Type: Microsoft Azure Event Hubs
    • Protocol: Microsoft Azure Event Hubs
    • Log Source Identifier: MDC-Alerts

Protocol Configuration

Parameter Value
Event Hub Namespace mdc-alerts-qradar.servicebus.windows.net
Event Hub Name defender-alerts
SAS Policy Name qradar-listen
SAS Policy Key (from Azure)
Consumer Group qradar-consumer

AQL Queries

MDC Alerts Overview

AQL 
SELECT 
    DATEFORMAT(starttime, 'yyyy-MM-dd HH:mm:ss') as "Time",
    LOGSOURCENAME(logsourceid) as "Source",
    "alertDisplayName" as "Alert Name",
    "severity" as "Severity",
    "compromisedEntity" as "Affected Resource",
    "status" as "Status"
FROM events
WHERE LOGSOURCENAME(logsourceid) ILIKE '%Azure%Defender%'
    AND starttime > NOW() - 24 HOURS
ORDER BY starttime DESC
LAST 100

High Severity Alerts

AQL 
SELECT 
    DATEFORMAT(starttime, 'yyyy-MM-dd HH:mm:ss') as "Time",
    "alertDisplayName" as "Alert",
    "compromisedEntity" as "Resource",
    "intent" as "MITRE Tactic"
FROM events
WHERE LOGSOURCENAME(logsourceid) ILIKE '%Azure%Defender%'
    AND "severity" IN ('High', 'Critical')
    AND starttime > NOW() - 7 DAYS
ORDER BY starttime DESC

Alert Count by Severity

AQL 
SELECT 
    "severity" as "Severity",
    COUNT(*) as "Alert Count"
FROM events
WHERE LOGSOURCENAME(logsourceid) ILIKE '%Azure%Defender%'
    AND starttime > NOW() - 30 DAYS
GROUP BY "severity"
ORDER BY "Alert Count" DESC

Top Affected Resources

AQL 
SELECT 
    "compromisedEntity" as "Resource",
    COUNT(*) as "Alert Count",
    UNIQUECOUNT("alertDisplayName") as "Unique Alert Types"
FROM events
WHERE LOGSOURCENAME(logsourceid) ILIKE '%Azure%Defender%'
    AND starttime > NOW() - 7 DAYS
GROUP BY "compromisedEntity"
ORDER BY "Alert Count" DESC
LAST 10

MITRE ATT&CK Distribution

AQL 
SELECT 
    "intent" as "MITRE Tactic",
    COUNT(*) as "Alert Count"
FROM events
WHERE LOGSOURCENAME(logsourceid) ILIKE '%Azure%Defender%'
    AND starttime > NOW() - 30 DAYS
    AND "intent" IS NOT NULL
GROUP BY "intent"
ORDER BY "Alert Count" DESC

Custom Detection Rules

High Severity MDC Alert Rule

  1. Go to Offenses ‚Üí Rules
  2. Click Actions ‚Üí New Event Rule
  3. Configure:
    • Rule Name: MDC High Severity Alert
    • Test Group: Events
  4. Add tests:
    • when the event matches Log Source Type = Microsoft Azure Event Hubs
    • and when severity is one of High, Critical
  5. Set response: Create offense

Multiple Alerts Same Resource

AQL (Building Block) 
-- Use as building block for correlation rule
SELECT "compromisedEntity"
FROM events
WHERE LOGSOURCENAME(logsourceid) ILIKE '%Azure%Defender%'
    AND starttime > NOW() - 1 HOURS
GROUP BY "compromisedEntity"
HAVING COUNT(*) >= 3

Troubleshooting

  • Verify Event Hub connection string
  • Check consumer group name
  • Confirm continuous export is enabled in MDC
  • Test network connectivity to Azure
  • Check QRadar log source status
  • Update Azure DSM to latest version
  • Check DSM configuration
  • Review sample events in payload viewer
  • Create custom properties if needed
  • Increase Event Hub throughput units
  • Add more partitions
  • Check QRadar event processor load
  • Consider dedicated consumer group
