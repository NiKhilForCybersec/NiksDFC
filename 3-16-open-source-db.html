Defender for Open-Source Databases

L1 Fundamentals L2 Technical 12 min read

Microsoft Defender for open-source relational databases provides threat protection for Azure Database for MySQL, PostgreSQL, and MariaDB. It detects anomalous activities indicating potential database threats.

Supported Services
  • Azure Database for MySQL (Single Server & Flexible Server)
  • Azure Database for PostgreSQL (Single Server & Flexible Server)
  • Azure Database for MariaDB

Enablement

Azure CLI 
# Enable Defender for open-source databases at subscription level
az security pricing create \
    --name OpenSourceRelationalDatabases \
    --tier Standard

# Verify enablement
az security pricing show --name OpenSourceRelationalDatabases

# List protected database servers
az mysql server list --query "[].{Name:name,RG:resourceGroup}" -o table
az postgres server list --query "[].{Name:name,RG:resourceGroup}" -o table
PowerShell 
# Enable Defender for open-source databases
Set-AzSecurityPricing -Name "OpenSourceRelationalDatabases" -PricingTier "Standard"

# Verify
Get-AzSecurityPricing | Where-Object { $_.Name -eq "OpenSourceRelationalDatabases" }

# List MySQL servers
Get-AzMySqlServer | Select-Object Name, ResourceGroupName, Location

# List PostgreSQL servers
Get-AzPostgreSqlServer | Select-Object Name, ResourceGroupName, Location
HCL (Terraform) 
resource "azurerm_security_center_subscription_pricing" "osdb" {
  tier          = "Standard"
  resource_type = "OpenSourceRelationalDatabases"
}

Alert Types

Alert Severity MITRE Tactic Description
Brute force attack High Credential Access Multiple failed login attempts from same IP
Successful brute force Critical Initial Access Successful login after brute force attempt
Suspected SQL injection High Initial Access Potential SQL injection in query
Access from unusual location Medium Initial Access Login from new geographic location
Access from suspicious IP Medium Initial Access Connection from known malicious IP
Access from Azure data center Low Initial Access Unexpected access from Azure IP range

Investigation Procedures

Brute Force Attack

Investigation Steps:

  1. Identify source IP and targeted user accounts
  2. Check if any login succeeded
  3. Review timeline and attack volume
  4. Determine if IP is known malicious
KQL 
// MySQL/PostgreSQL failed login attempts
AzureDiagnostics
| where ResourceProvider == "MICROSOFT.DBFORMYSQL" 
    or ResourceProvider == "MICROSOFT.DBFORPOSTGRESQL"
| where Category == "MySqlAuditLogs" or Category == "PostgreSQLLogs"
| where Message contains "Access denied" or Message contains "authentication failed"
| summarize FailedAttempts = count() by ClientIP = client_ip_s, User = user_s, bin(TimeGenerated, 5m)
| where FailedAttempts > 10
| order by FailedAttempts desc

Response:

  • Block attacking IP in firewall rules
  • If login succeeded, rotate affected credentials
  • Review database for unauthorized changes
Azure CLI 
# Block IP in MySQL firewall
az mysql server firewall-rule delete \
    --server-name "mysql-server" \
    --resource-group "db-rg" \
    --name "allow-attacker-ip"

# Or for PostgreSQL
az postgres server firewall-rule delete \
    --server-name "postgres-server" \
    --resource-group "db-rg" \
    --name "allow-attacker-ip"
SQL Injection Attempt

Investigation:

  1. Review the query that triggered the alert
  2. Identify the application making the query
  3. Check if injection was successful
  4. Review for data access or modification
KQL 
// Find suspicious queries
AzureDiagnostics
| where ResourceProvider == "MICROSOFT.DBFORMYSQL"
| where Category == "MySqlAuditLogs"
| where sql_text_s contains "UNION" 
    or sql_text_s contains "OR 1=1"
    or sql_text_s contains "--"
    or sql_text_s contains "DROP"
| project TimeGenerated, client_ip_s, user_s, db_s, sql_text_s

Response:

  • Block source IP
  • Review application code for vulnerabilities
  • Implement parameterized queries
  • Check database integrity

Security Best Practices

Database Security Checklist

Enable Audit Logging

Azure CLI 
# Enable diagnostic logging for MySQL
az monitor diagnostic-settings create \
    --name "mysql-audit-logs" \
    --resource "/subscriptions/.../providers/Microsoft.DBforMySQL/servers/mysql-server" \
    --workspace "/subscriptions/.../providers/Microsoft.OperationalInsights/workspaces/your-workspace" \
    --logs '[{"category": "MySqlAuditLogs", "enabled": true}]'

# Enable for PostgreSQL
az monitor diagnostic-settings create \
    --name "postgres-logs" \
    --resource "/subscriptions/.../providers/Microsoft.DBforPostgreSQL/servers/postgres-server" \
    --workspace "/subscriptions/.../providers/Microsoft.OperationalInsights/workspaces/your-workspace" \
    --logs '[{"category": "PostgreSQLLogs", "enabled": true}]'
Key Takeaways
  • Protects MySQL, PostgreSQL, and MariaDB on Azure
  • Detects brute force, SQL injection, and anomalous access
  • Pricing is ~$15/server/month
  • Enable at subscription level with OpenSourceRelationalDatabases plan
  • Use firewall rules and Private Endpoints for network security
  • Enable audit logging for investigation capability
  • Enforce SSL and strong authentication
  • Brute force followed by successful login is critical severity
