Governance Rules

L1 Fundamentals L2 Operations 12 min read Defender CSPM

Governance rules in Microsoft Defender for Cloud automate the assignment of security recommendations to resource owners with defined remediation timeframes. This transforms security findings into actionable tasks with accountability.

Requires Defender CSPM

Governance rules are a premium feature requiring Defender CSPM plan. The free tier includes basic recommendations but not automated governance workflows.

Creating Governance Rules

Create via Portal

  1. Azure Portal â†’ Defender for Cloud â†’ Environment settings
  2. Select subscription â†’ Governance rules
  3. Click "+ Create governance rule"
  4. Configure:
    • Name: Descriptive rule name
    • Scope: Where to apply (subscription/RG)
    • Priority: Lower number = higher priority
    • Conditions: Which recommendations to match
    • Owner: Email or resource tag
    • Remediation timeframe: 7-180 days by severity
  5. Enable email notifications if desired
  6. Save the rule
PowerShell (REST API) 
# Create governance rule via REST API
$subscriptionId = (Get-AzContext).Subscription.Id
$ruleName = "critical-findings-14-days"

$body = @{
    properties = @{
        displayName = "Critical Findings - 14 Day Remediation"
        description = "Assign critical findings to owners with 14-day SLA"
        isDisabled = $false
        rulePriority = 100
        sourceResourceType = "Assessments"
        conditionSets = @(
            @{
                conditions = @(
                    @{
                        property = "$.AssessmentProperties.Metadata.Severity"
                        value = "High"
                        operator = "Equals"
                    }
                )
            }
        )
        ownerSource = @{
            type = "ByTag"
            value = "SecurityOwner"
        }
        governanceEmailNotification = @{
            disableManagerEmailNotification = $false
            disableOwnerEmailNotification = $false
        }
        remediationTimeframe = "14.00:00:00"
        isGracePeriod = $true
        gracePeriod = "7.00:00:00"
    }
} | ConvertTo-Json -Depth 10

Invoke-AzRestMethod -Method PUT `
    -Path "/subscriptions/$subscriptionId/providers/Microsoft.Security/governanceRules/$ruleName`?api-version=2022-01-01-preview" `
    -Payload $body

Using Resource Tags for Owner Assignment

The most scalable approach is to assign owners based on resource tags. Ensure your tagging strategy includes an owner email tag like SecurityOwner or TeamEmail on all resources.

Monitoring Governance

KQL Queries

KQL 
// Governance assignments status
SecurityRecommendation
| where TimeGenerated > ago(7d)
| where isnotempty(Owner)
| summarize 
    arg_max(TimeGenerated, *) 
    by RecommendationName, AssessedResourceId
| summarize 
    TotalAssignments = count(),
    Overdue = countif(RemediationDueDate < now() and RecommendationState == "Unhealthy"),
    Completed = countif(RecommendationState == "Healthy")
    by Owner
| extend CompletionRate = round(100.0 * Completed / TotalAssignments, 1)
| order by Overdue desc
KQL 
// Overdue remediation items
SecurityRecommendation
| where TimeGenerated > ago(1d)
| where RecommendationState == "Unhealthy"
| where RemediationDueDate < now()
| project 
    RecommendationName,
    AssessedResourceId,
    Owner,
    Severity = RecommendationSeverity,
    DueDate = RemediationDueDate,
    DaysOverdue = datetime_diff('day', now(), RemediationDueDate)
| order by DaysOverdue desc
KQL 
// Remediation trends by severity
SecurityRecommendation
| where TimeGenerated > ago(30d)
| summarize 
    Unhealthy = countif(RecommendationState == "Unhealthy"),
    Healthy = countif(RecommendationState == "Healthy")
    by RecommendationSeverity, bin(TimeGenerated, 1d)
| extend RemediationRate = round(100.0 * Healthy / (Healthy + Unhealthy), 1)
| order by TimeGenerated asc, RecommendationSeverity
| render timechart

Key Takeaways
  • Governance rules require Defender CSPM plan
  • Rules automatically assign owners and due dates to recommendations
  • Use resource tags for scalable owner assignment
  • Set remediation SLAs based on severity (7/14/30/90 days)
  • Enable email notifications for new assignments and approaching due dates
  • Rules are processed by priority (lower number = higher priority)
  • Monitor governance metrics in the built-in workbook
  • Integrate with ITSM tools for enterprise workflows
