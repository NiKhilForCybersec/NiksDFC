Sentinel Integration Architecture

L2 Technical Architect 20 min read

Integrating Microsoft Defender for Cloud with Microsoft Sentinel creates a unified cloud-native SIEM+XDR solution. This integration enables centralized alert management, advanced correlation, automated response, and comprehensive threat hunting across your cloud estate.

Why Integrate?
  • Correlate MDC alerts with other security signals (identity, endpoint, network)
  • Leverage Sentinel's SOAR capabilities for automated response
  • Use KQL for advanced hunting and custom detections
  • Create unified dashboards across all security telemetry

Sync Behavior

When bi-directional sync is enabled:

  • Dismissing an alert in Sentinel marks it as dismissed in MDC
  • Resolving an alert in MDC updates the status in Sentinel
  • Sync happens in near real-time (usually within minutes)

Connector Options

Microsoft Defender for Cloud (Native)

Recommended for most scenarios

  • One-click enablement in Sentinel
  • Automatic subscription discovery
  • Bi-directional sync supported
  • Includes built-in analytics rules
Azure CLI 
# Enable via CLI (alternative to portal)
az sentinel data-connector create \
    --resource-group "rg-sentinel" \
    --workspace-name "law-sentinel" \
    --data-connector-id "MicrosoftDefenderForCloud" \
    --kind "AzureSecurityCenter" \
    --properties '{
        "subscriptionId": "your-subscription-id",
        "dataTypes": {
            "alerts": { "state": "Enabled" }
        }
    }'

Tenant-Based Licensing (Preview)

For organizations with multiple subscriptions:

  • Single connection covers all subscriptions in tenant
  • Automatic onboarding of new subscriptions
  • Simplified management
  • Requires specific licensing

Subscription-Based

For granular control:

  • Select specific subscriptions to connect
  • Different configurations per subscription
  • Useful for staged rollouts
  • More management overhead

Unified XDR + SIEM Operations

When MDC is integrated with Sentinel alongside other Microsoft 365 Defender products, you achieve unified security operations across endpoints, identities, cloud workloads, and data.

Integration Best Practices

1. Enable Bi-Directional Sync

Always enable bi-directional sync to prevent duplicate work. When SOC analysts dismiss alerts in Sentinel, those changes propagate back to MDC automatically.

2. Consider Recommendation Volume

Security recommendations can generate high data volume. Consider:

  • Only enabling for compliance reporting use cases
  • Using Azure Resource Graph for real-time recommendation queries
  • Setting appropriate retention to manage costs
3. Create Custom Analytics Rules

Leverage Sentinel's analytics capabilities to correlate MDC alerts with other signals:

  • Correlate cloud alerts with identity events from Azure AD
  • Link server threats with endpoint telemetry from MDE
  • Create custom severity rules based on asset criticality
4. Build Automated Response Playbooks

Use Logic Apps playbooks to automate responses:

  • Auto-isolate VMs on critical malware alerts
  • Create ServiceNow tickets for compliance findings
  • Enrich alerts with threat intelligence lookups
  • Notify teams via Teams/Slack on high severity alerts
Key Takeaways
  • MDC + Sentinel integration creates unified CNAPP + SIEM operations
  • Bi-directional sync keeps alert status synchronized across platforms
  • SecurityAlert, SecureScores, and SecureScoreControls are core tables
  • Consider data volume when enabling SecurityRecommendation sync
  • Use Sentinel's analytics and SOAR capabilities for advanced correlation and response
  • Combine with M365 Defender for complete XDR coverage
